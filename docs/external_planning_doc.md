# ペット用品ECサイト「星野のペット用品」）」外部計画書

## 1. プロジェクト概要
本プロジェクトは、日本国内市場をターゲットとした新規ペット用品販売事業の基盤となるECサイトを構築するものである。個人事業としてスモールスタートを切ることを前提としつつ、競合他社にはない「AIによる試着体験」と「カスタマイズ性」を強みとした、プレミアムなショッピング体験の提供を目指す。

## 2. ターゲットユーザー
- 日本国内のペット飼育者（主に犬・猫）。
- コスパを優先し、低〜中価格帯で購入する層。
- 「自分のペットに似合うかどうか」を重視する層。


## 3. システム構成・画面一覧（サイトマップ）
### 3.1 ユーザー向け画面（フロントエンド）
- トップページ（商品一覧・特集）
- 商品検索・カテゴリ別一覧ページ
- 商品詳細ページ（AI試着プレビュー、ワッペンカスタマイズ）
- ショッピングカート・決済画面
- マイページ（注文履歴、ペットプロフィール・画像管理）

### 3.2 管理者向け画面（バックエンド・管理画面）
- ダッシュボード（売上・注件数サマリー、主要KPIの可視化）
- アクセス解析・マーケティング（訪問者数、流入経路、離脱率、人気商品ランキング、AI試着利用データ）
- 商品管理（新規登録、在庫管理、ワッペンアセット管理）
- 注文管理（ステータス更新：未発送→発送済み等）
- 顧客管理（会員情報、購入履歴照会）

## 4. 主要機能
### 4.1 基本機能（ECサイト標準）
- **商品閲覧・検索**: カテゴリ、キーワード、タグ（犬種・猫種向けなど）による絞り込み。
- **カート・決済**: KOMOJUを利用したセキュアな決済処理。
- **ユーザー認証・管理**: 会員登録、ログイン、パスワードリセット、ペットプロフィールの詳細登録（名前、種、性別、体重、体長）。
- **CRM機能**: 購入履歴に基づくレコメンド、メール配信（クーポン、カゴ落ちリマインド）。
- **レビュー機能**: 購入者による評価とコメント。

### 4.2 特別機能：AIペット試着（Virtual Try-on）
- **画像アップロード**: ユーザーの情報として,ペットの画像欄に自分のペット（犬・猫等）の画像をアップロード。
アップロード時は正面，斜め45度，横の三種類の写真を要求する．
- **AI合成生成**: アップロードされたペットの画像と、選択した商品（服、首輪、アクセサリー等）を自然に合成。合成する画像はデフォルトで斜め45度の1枚である。更に生成を押すとさらに正面，横からの画像も生成される．ただし合計4枚を超えない．
- **プレビュー表示**: 生成された画像を商品詳細ページで確認。そのまま購入意欲を高める。

### 4.3 特別機能②：ワッペン付けカスタマイズ
- **UI操作**: 商品詳細ページにおいて、スマホのタッチ操作に最適化されたドラッグ＆ドロップ（またはタップ）で、商品画像上の指定エリアにワッペンを配置。
- **データ保存**: 配置したワッペンのIDと座標情報（X, Y）をJSON形式等でカート情報に保持し、注文データの一部として管理者へ送信する。

### 4.4 データ解析・マーケティング機能（Google Analytics風）
- **トラフィック分析**: 日別・月別の訪問者数（UU/PV）、セッション維持時間、直帰率の計測。
- **コンバージョン計測**: 商品詳細からのカート投入率、AI試着体験後の購入転換率の可視化。
- **行動ログトラッキング**: ユーザーがどのワッペンを好んで試着しているか、どのペット種（犬/猫）のアクセスが多いか等の属性・行動分析。
- **レポート出力**: 売上予測や在庫補充タイミングの示唆を含む、管理者向け自動生成レポート。

## 5. 非機能要件
- **モバイル最適化**: スマホからのアクセス（想定80%以上）を最優先としたモバイルファースト設計。
- **パフォーマンス**: AI画像生成の待機時間（APIレスポンス）中は、スケルトンスクリーンやローディングアニメーションを表示し、UXの低下を防ぐ。
- **セキュリティ**: 個人情報（顧客情報・ペット画像）の適切な保護。決済情報は自社サーバーに保持せずKOMOJU側へ委譲する。

## 6. 技術スタック
個人開発のスピードとモダンな体験を両立しつつ、ECとしての堅牢性を確保する構成。

- **フロントエンド**: Next.js (React), Vanilla CSS
    - SEO対策、高速なページ遷移、リッチなUI（マイクロインタラクション）の実現。
- **バックエンド**: Python 3.13, FastAPI
    - 高速なAPI開発、型安全性、OpenAPI(Swagger)によるドキュメント自動生成。
- **データベース**: PostgreSQL (Supabase等のPaaS利用を想定)
    - ECサイトのトランザクション（決済・在庫確保等）の安全性を確保するため、SQLiteからRDBMSへアップグレード。
- **外部API**:
    - 決済: KOMOJU (テスト環境: `pk_test_dummy_...`, `sk_test_dummy_...`)
    - 画像合成: 外部AI画像生成API (ダミーキーでモック実装から開始)
- **インフラ**: 自社Ubuntu VPS (Frontend & Backend 一体運用)
    - CaddyをWebサーバー/リバースプロキシとして採用。HTTPS化（Let's Encrypt）の完全自動運用を最大限活用。
    - シンプルなCaddyfile管理により、Next.jsとFastAPIの共存設定を最小限の工数で維持する。

## 5. 外部サービス連携
### 5.1 支払い方法 (KOMOJU)
日本の多様な決済ニーズに応えるため、[KOMOJU](https://ja.doc.komoju.com/)を統合する。

- **対応決済**: クレジットカード、コンビニ決済、PayPay、銀行振込等。
- **開発環境**: 登録前のため、APIキーはダミーを使用し、サンドボックス環境を想定した実装を行う。
    - Public Key: `pk_test_dummy_123456789`
    - Secret Key: `sk_test_dummy_987654321`

## 6. デザインコンセプト
- **プレミアム感**: 清潔感のある白を基調とし、アクセントカラーに深みのあるゴールドや落ち着いたネイビーを採用。
- **マイクロインタラクション**: カート投入時、ワッペン配置時、画像生成完了時に、高級感を感じさせるスムーズなアニメーションを実装。
